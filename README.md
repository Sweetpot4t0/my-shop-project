# 감자몰 (Potato Mall)
스프링 부트와 JPA를 활용한 감자 전문 쇼핑몰 프로젝트입니다.

## 주요 기능
- **사용자**: 회원가입/로그인, 상품 주문, 마이페이지(주문 내역 확인)
- **관리자**: 전용 대시보드, 전체 주문 현황 관리 (배송 상태 확인 가능)

## Tech Stack
- **Backend**: Java 17, Spring Boot 3.x, Spring Data JPA
- **Database**: PostgreSQL 18.3
- **Frontend**: HTML5, CSS3, JavaScript (Vanilla JS)

## 💡 주요 해결 과제 (Troubleshooting)
##1. JPA N+1 문제 해결 (Fetch Join)
문제: 관리자 페이지에서 전체 주문 목록을 불러올 때, 주문자 정보를 가져오기 위해 주문 수만큼 추가 쿼리가 발생하는 비효율이 있었습니다.

해결: JOIN FETCH를 사용하여 연관된 엔티티를 한 번의 쿼리로 함께 조회하도록 최적화했습니다.

##2. JSON 순환 참조 및 직렬화 이슈 해결
문제: Order와 Member 객체 간의 양방향 참조로 인해 JSON 변환 시 무한 루프가 발생하거나 불필요한 데이터가 노출되었습니다.

해결: @JsonIgnoreProperties와 **DTO(OrderRequestDto, OrderResponseDto)**를 도입하여 필요한 데이터만 프론트엔드로 정확하게 전달하도록 개선했습니다.

##3. BCrypt를 활용한 보안 강화 (신규)
문제: 사용자 비밀번호를 평문으로 저장할 경우 보안에 매우 취약한 상태였습니다.

해결: BCryptPasswordEncoder를 도입하여 비밀번호 저장 시 암호화(Hashing)를 적용했습니다. 로그인 시에도 matches() 메서드를 통해 안전하게 검증하도록 로직을 구현했습니다.

##4. 서비스 레이어 분리 및 아키텍처 최적화 (신규)
문제: 컨트롤러에 비즈니스 로직이 집중되어 코드 유지보수가 어렵고 역할이 모호했습니다.

해결: MemberService, OrderService 등 서비스 레이어를 분리하여 비즈니스 로직을 캡슐화했습니다. 이를 통해 컨트롤러는 요청과 응답에만 집중하도록 구조를 탄탄하게 다졌습니다.

##5. 세션 기반 권한 제어 및 인터셉터 적용
문제: 비로그인 사용자나 권한이 없는 사용자가 중요 페이지에 접근할 수 있는 위험이 있었습니다.

해결: LoginCheckInterceptor를 구현하고 세션 정보를 활용하여 Role 기반 접근 제어를 적용했습니다. 일반 사용자와 관리자의 UI/UX를 분리하여 보안과 사용자 경험을 동시에 강화했습니다.
